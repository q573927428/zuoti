# 币安API配置说明

## 当前状态

目前系统使用的是币安模拟交易API密钥，可能会遇到以下问题：

### 常见错误

```
ERROR 获取账户余额失败: binance {"code":-2015,"msg":"Invalid API-key, IP, or permissions for action."}
```

这个错误说明当前的API密钥权限不足或已过期。

## 解决方案

### 方案1：使用币安测试网（推荐用于学习）

1. 访问币安测试网：https://testnet.binance.vision/
2. 注册并获取测试网API密钥
3. 在测试网会自动分配虚拟资金
4. 更新 `.env` 文件中的API密钥

### 方案2：使用真实币安账户（用于实盘）

1. 登录币安官网：https://www.binance.com/
2. 进入API管理页面创建新的API密钥
3. 设置API权限（需要启用现货交易权限）
4. 将 `isTestnet` 设置为 `false`
5. 更新 `.env` 文件中的API密钥

**⚠️ 注意事项：**
- 真实交易会使用真实资金，请谨慎操作
- 建议先用小额资金测试
- 不要将API密钥分享给任何人
- 定期更换API密钥

### 方案3：忽略余额错误（临时方案）

即使余额获取失败，系统的其他功能仍然可以正常使用：
- ✅ 实时振幅分析
- ✅ 手动交易
- ✅ 调试日志
- ✅ 测试连接

余额显示会显示为0，但不影响交易功能。

## 配置文件位置

`.env` 文件内容：
```bash
BINANCE_API_KEY=你的API密钥
BINANCE_SECRET=你的密钥Secret
```

## 测试连接

点击界面上的"测试连接"按钮，可以验证API配置是否正确。

## 技术支持

如果遇到其他问题，请查看：
1. 调试日志面板 - 查看详细错误信息
2. 浏览器控制台 - 查看网络请求
3. 服务器控制台 - 查看后端日志
